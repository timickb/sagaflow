# Saga Consistency Orchestrator

Представляет собой оркестратор распределенных транзакций с обязательным слоем верификации.
Верификация работает как проверка консистентности данных в гетерогенных хранилищах.
В случае обнаружения расхождений происходит откат саги с использованием компенсационных шагов
и запросов в хранилища (заранее определенных в описании саги), восстанавливающих консистентность данных.
Таким образом оркестратор гарантирует end-to-end консистентность всего бизнес-факта, фиксируемого сагой.

## Архитектура

Оркестратор представляет собой сервис на go, состоящий из компонентов:

* Парсер описаний саг (DSL) во внутреннюю графовую модель
* Асинхронный обработчик экземпляров саг
* Consumer для Kafka, обрабатывающий результаты фиксации локальных транзакций обработчиками
* gRPC, предоставляющий API для управления сагами

## Описание саги

Сага описывается в формате YAML. Пример: `sagas/process-order.1.yaml`

Описание состоит из двух частей: заголовок и список шагов.

## Матрица правил переходов экземпляра для каждого типа шага

### Шаг типа Action (прямая локальная транзакция)

| Исход шага | Что означает                                                      | Что делает движок                                                                                                                                      |
|------------|-------------------------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| committed  | Локальная транзакция успешно зафиксирована                        | Переходит по `on.committed`                                                                                                                            |
| rejected   | Не соблюдены бизнес-предусловия для выполнения шага; шаг отклонен | Переходит по `on.rejected`                                                                                                                             |
| failed     | Техническая ошибка, невозможно зафиксировать транзакцию           | Если еще остались повторные попытки - использует их; иначе переходит по on.failed. Если `on.failed` не задан - завершает экземпляр со статусом FAILED. |
| timeout    | Результат шага не пришел за ожидаемое время                       | Помечает шаг FAILED с `effect_state=UNKNOWN`. Если задан `on.failed` - идет по нему, иначе запускает reconciliation                                    |

### Шаг типа Verify (сверка состояния данных в системах)

| Исход шага | Что означает                                      | Что делает движок                                                                                                                                            |
|------------|---------------------------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------|
| matched    | Инвариант соблюден                                | Переходит по `on.passed`                                                                                                                                     |
| unmatched  | Инвариант нарушен                                 | Переходит по `on.failed`                                                                                                                                     |
| timeout    | Проверка не завершилась вовремя                   | Если еще остались повторные попытки - использует их; иначе переходит по on.failed. Если `on.failed` не задан - завершает экземпляр со статусом INCONSISTENT. |
| failed     | Техническая ошибка, невозможно получить результат | То же самое, что при `on.failed`                                                                                                                             |

### Шаг типа Reconcile (исправление данных в системах)

| Исход шага | Что означает                  | Что делает движок                                                                                |
|------------|-------------------------------|--------------------------------------------------------------------------------------------------|
| committed  | Исправление выполнено успешно | Переходит по `on.committed`                                                                      |
| failed     | Исправление не удалось        | Переходит по `on.failed`; если переход не задан - завершение экземпляра со статусом INCONSISTENT |
| timeout    | Система не ответила вовремя   | Если еще остались повторные попытки - использует их; иначе см. исход failed                      |

### Шаг типа Compensate (компенсирующая локальная транзакция = откат шага типа Action)

| Исход шага | Что означает                                    | Что делает движок                                                                                                                                           |
|------------|-------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|
| committed  | Компенсирующая транзакция успешно зафиксирована | Если есть предыдущие шаги - переход к их компенсации; иначе завершение экземпляра со статусом COMPENSATED                                                   |
| failed     | Не удалось выполнить компенсацию                | Если еще остались повторные попытки - использует их; иначе переходит по on.failed. Если `on.failed` не задан - завершает экземпляр со статусом INCONSISTENT |
| timeout    | Результат шага не пришел за ожидаемое время     | Помечает шаг FAILED с `effect_state=UNKNOWN`. Если задан `on.failed` - идет по нему, иначе завершает экземпляр со статусом INCONSISTENT                     |

### Шаг типа Terminal

| Исход шага          | Что означает                                         | Что делает движок                        |
|---------------------|------------------------------------------------------|------------------------------------------|
| result=COMPLETED    | Экземпляр успешно завершен                           | Переводит экземпляр в статус COMPLETED   |
| result=FAILED       | Экземпляр завершен с ошибкой                         | Переводит экземпляр в статус FAILED      |
| result=COMPENSATED  | Выполнен откат, бизнес-цель не достигнута            | Переводит экземпяр в статус COMPENSATED  |
| result=INCONSISTENT | Не удалось доказать или восстановить консистентность | Переводит экземпяр в статус INCONSISTENT |

### Default semantics

| Ситуация                                                  | Поведение по умолчанию                         |
|-----------------------------------------------------------|------------------------------------------------|
| Action завершился как failed, но `on.failed` не задан     | Завершение экземпляра со статусом FAILED       |
| Verify завершился как failed, но `on.failed` не задан     | Завершение экземпляра со статусом INCONSISTENT |
| Reconcile завершился как failed, но `on.failed` не задан  | Завершение экземпляра со статусом INCONSISTENT |
| Compensate завершился как failed, но `on.failed` не задан | Завершение экземпляра со статусом INCONSISTENT |
